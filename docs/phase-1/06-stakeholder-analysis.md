# Stakeholder Analysis

| Stakeholder | Goals | Needs | Risks |
| --- | --- | --- | --- |
| Organization Owner | Govern platform use | Tenant settings, branding, audit | Misconfiguration |
| Admin | Manage users and organization data | RBAC, departments, batches, trainers | Excessive manual work |
| Trainer | Create and run assessments | Question bank, templates, live dashboard | Complex builder |
| Candidate | Complete assessment fairly | Clear instructions, autosave, mobile support | Network interruption |
| L&D Leader | Measure skill growth | Reports, weak-area analysis, trends | Incomplete analytics |
| HR | Track eligibility and history | Candidate records and exports | Data mismatch |
| Security Team | Protect data and assessment integrity | Audit, encryption, rate limits | Cheating or leakage |
| IT/Ops | Deploy and maintain | Docker, logs, metrics, backups | Operational complexity |
| External Systems | Exchange events | REST APIs, webhooks | Integration failures |

## Communication Plan

- Weekly implementation review during active phases.
- Security review before candidate flow release.
- Data model review before Phase 2 approval.
- UAT with admin, trainer, and candidate personas before production release.
